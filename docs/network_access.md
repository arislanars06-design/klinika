# Тармоққа очиш — бошқа компьютер ва телефонлардан фойдаланиш

Клиника дастури **веб дастур** бўлгани учун битта компьютерда ишга туширсангиз, ўша ишлаётган компьютер билан бир Wi-Fi ёки локал тармоқдаги **исталган қурилма** (телефон, планшет, бошқа компьютер) браузер орқали кириши мумкин. Word ўрнатиш, махсус илова ўрнатиш **керак эмас** — фақат браузер (Chrome, Safari, Firefox, Edge — исталгани).

Қуйида қадам-бақадам йўриқнома.

---

## 1. Клиникадаги битта «сервер» компьютерни танланг

Барча маълумотлар шу битта компьютерда сақланади. Одатда бу — ресепшн ёки ҳамширалик жойидаги компьютер. Уни **«сервер»** деб атаймиз.

Талаблар:
- Windows 10 / 11
- Клиника Wi-Fi'га уланган (симли ёки симсиз, аҳамияти йўқ)
- Дастур одатдагидек ўрнатилган (`.venv`, `pip install`, база яратилган)

## 2. `start-network.bat`'ни ишлатинг

Клиника папкасини очинг → **`start-network.bat`**'ни икки марта босинг (`start.bat` эмас — тармоққа очмайди).

Терминал очилади ва тахминан шундай кўринади:

```
================================================================
  KLINIKA WEB - tarmoqqa ochilgan (LAN access)
================================================================

  Bu kompyuterda ochish uchun:
    http://localhost:8000

  Bir xil Wi-Fi'dagi telefon / boshqa kompyuterdan ochish uchun
  quyidagi manzillardan BIRINI brauzerga kiriting:

    http://192.168.1.42:8000
    http://192.168.10.5:8000
```

**Ёзиб олинг ёки экрандан фотога тортиб қўйинг** — бошқа қурилмаларга шу манзил керак.

## 3. Windows Firewall'га рухсат беринг

Биринчи марта `start-network.bat`'ни ишга туширганингизда Windows «**Do you want to allow Python to communicate on these networks?**» деб сўрайди.

- ✅ «**Private networks**» ва «**Public networks**» — иккаласига ҳам белги қўйинг
- ✅ «**Allow access**» тугмасини босинг

Агар сўрамай ўтиб кетган бўлса — 6-бўлимга қаранг.

## 4. Телефон/бошқа компьютердан кириш

### Телефон (Android/iPhone)

1. Телефон **шу Wi-Fi'га** уланганлигини текширинг.
2. Chrome ёки Safari'ни очинг.
3. Юқорида кўрсатилган манзилни адрес қаторига ёзинг, масалан:

   ```
   http://192.168.1.42:8000
   ```

   > ⚠️ `http` бўлади, `https` **эмас**. Порт (`:8000`) ҳам мажбурий.

4. Логин саҳифаси очилади → одатдаги логин/паролингиз билан киринг.

### Бошқа компьютер (Windows, Mac, Linux)

Худди шундай — исталган браузер, шу манзил, шу логин.

## 5. Ҳар кимга ўзининг ҳисоби

Тармоққа очганингиздан кейин **ҳар бир ходимга** алоҳида **логин/парол** яратинг. Бир парол ҳаммага деб қўяверманг!

**Созламалар → Ходимлар (WebUsers)** бўлимида:
- Ҳар шифокор учун — доктор роли
- Ресепшн ходими учун — reception роли
- Кассир учун — cashier роли
- Администратор фақат сиз бўлинг

Бу орқали:
- Ким қайси бемор билан ишлаганлиги логга ёзилади
- Ходим ишдан кетса — фақат унинг ҳисобини ўчирасиз, парол ўзгартирмайсиз

## 6. Диагностика — ишламаса нима қилиш

### «Manzil topilmadi» / «This site can't be reached»

- Телефон **айнан ўша Wi-Fi'да**ми? (мобил интернет ЭМАС)
- Сервер компьютерда `start-network.bat` ишлаяптими? (терминал очиқми?)
- Манзилни **аниқ** ёзганмисиз? (`http://` + IP + `:8000`)

### IP манзилни билмайман

Сервер компьютерда PowerShell очинг ва ёзинг:

```powershell
ipconfig | findstr IPv4
```

Кўрсатилган рақамлардан бирини (одатда `192.168.` билан бошланадиган) телефонга ёзинг.

### Windows Firewall блоклаяпти

PowerShell'ни **administrator сифатида** очинг ва:

```powershell
New-NetFirewallRule -DisplayName "Klinika Web" -Direction Inbound -LocalPort 8000 -Protocol TCP -Action Allow
```

Бу битта марта, доимий рухсат.

### Порт 8000 бошқа дастур томонидан ишлатилган

Бошқа порт танланг, масалан 8080:

`start-network.bat`'ни таҳрирлаш ўрнига, `.env` файлига қўшинг:

```
CLINIC_WEB_PORT=8080
```

Ва бошқа қурилмаларда `http://<IP>:8080` дан кириш.

## 7. Хавфсизлик

Тармоққа очганингизда:

- ✅ **Ҳар бир ходимга алоҳида парол** (5-бўлим).
- ✅ Клиника Wi-Fi'га **фақат ходимлар** киришини таъминланг (WPA2/WPA3 паролини кучли қилинг, беморларга алоҳида «гест» Wi-Fi беринг).
- ⚠️ **Интернетга очманг** (роутерда port forwarding **ҚИЛМАНГ**). Ҳозирги ҳимоя фақат локал тармоқ учун — очиқ интернетга чиқариш учун HTTPS, IP оқ рўйхати, тунелинг керак бўлади. Бунга алоҳида ёрдам керак бўлса — сўранг.
- 💾 **Резерв нусха** — Созламалар → Захира (backup) орқали ҳар куни / ҳафтада бир маротаба.

## 8. Клиникадан ташқарида (уйдан, йўлдаги) ишлаш?

Бу учун **алоҳида ечим** керак. Тавсиялар:

1. **Cloudflare Tunnel** (бепул) — сервер компьютерингизни хавфсиз тарзда интернетга тешик очмасдан очади. `cloudflared` дастури + Cloudflare аккаунт.
2. **Tailscale** — «шахсий VPN». Клиника компьютери ва уйдаги телефон бир виртуал тармоқда бўлади.
3. **VPS'га деплой** (энг тўғри узоқ муддатли ечим) — маълумотлар булутли серверда, ҳар доим онлайн, барча жойдан кириш.

Бу ечимларда ёрдам керак бўлса алоҳида сўранг.

## Қисқа хулоса

| Жой | Қадам |
|---|---|
| Сервер компьютер | `start-network.bat`'ни ишга туширинг. IP манзилини ёзиб олинг. |
| Windows Firewall | «Allow access» тугмасини босинг (биринчи маротаба). |
| Ҳар бир телефон/компьютер | Худди ўша Wi-Fi'га уланинг → браузерда `http://<IP>:8000` очинг. |
| Ҳар бир ходим | Ўз логин/паролингиз билан киринг. |
